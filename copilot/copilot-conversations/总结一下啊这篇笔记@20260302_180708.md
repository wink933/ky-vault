## 📌 **标题：折半查找（二分查找）**

### ✅ 适用前提：
- 只适用于**关键字有序的顺序表**。
- 要求数据结构支持**随机访问**（即数组），不适用于链式存储。

---

## 🔍 基本思想：

1. 将目标值 `key` 与当前查找区间的**中间元素**比较：
   - 若相等 → 查找成功，返回位置。
   - 若 `key < 中间元素` → 在左半部分继续查找。
   - 若 `key > 中间元素` → 在右半部分继续查找。
2. 每次缩小范围（区间长度减半），直到找到目标或确定不存在。

> ⚠️ **关键点**：每次取中间位置时，必须统一使用“向下取整”或“向上取整”，不能混用！

---

## 💡 算法实现（C语言伪代码）：

```c
int Binary_Search(SSTable L, ElemType key) {
    int low = 0, high = L.TableLen - 1, mid;
    while (low <= high) {
        mid = (low + high) / 2;           // 取中间位置（向下取整）
        if (L.elem[mid] == key)
            return mid;                   // 找到，返回索引
        else if (L.elem[mid] > key)
            high = mid - 1;               // 在左半部分查找
        else
            low = mid + 1;                // 在右半部分查找
    }
    return -1;                            // 查找失败
}
```

---

## 📈 算法过程示例：

以有序表 `{7,10,13,16,19,29,32,33,37,41,43}` 为例：
- 查找 `key=11` → 不在表中，最终返回 `-1`
- 查找 `key=32` → 成功找到，位置为索引 `6`

> 🖼️ 图解过程可通过中间指针移动逐步缩小范围（见笔记中的图示）。

---

## 🌲 判定树结构：

- **圆形结点**：表示表中存在的记录。
- **方形结点**：失败结点，代表查找失败的区间。
- 满足“左子树 < 根 < 右子树”的性质 → 是一棵**平衡二叉搜索树**。
- 若有 `n` 个元素，则判定树包含：
  - `n` 个圆形非叶节点（成功结点）
  - `n+1` 个方形叶节点（失败结点）

---

## 📊 平均查找长度（ASL）：

### ✅ 等概率查找时的平均查找长度公式：

$$
ASL = \frac{1}{n} \sum_{i=1}^{n} l_i = \frac{n+1}{n} \log_2(n+1) - 1 \approx \log_2(n+1) - 1
$$

### 📌 示例（n=11）：

- 查找成功 ASL = (1×1 + 2×2 + 3×4 + 4×4)/11 = **3**
- 查找失败 ASL = (3×4 + 4×8)/12 = **11/3 ≈ 3.67**

---

## ⏱️ 时间复杂度：

- 最坏情况：比较次数不超过树高 → $ h = \lceil \log_2(n+1) \rceil $
- 因此，**时间复杂度为 O(log₂n)** —— 高效！

> 📈 与顺序查找（O(n)）相比，折半查找在大数据量下效率显著提升。

---

## ⚠️ 注意事项：

1. **必须有序**：数据需按关键字升序或降序排列。
2. **仅支持顺序存储结构**：如数组。链表无法随机访问中间元素 → 不适用。
3. **取整方式统一**：若用 `(low + high) / 2`，则默认向下取整；若想向上取整，可用 `((low + high + 1) / 2)`。

---

## 🧠 总结一句话：

> 折半查找是一种高效的有序表查找算法，利用“分治思想”每次将搜索范围减半，时间复杂度为 O(log₂n)，但要求数据必须存储在数组中且已排序。
