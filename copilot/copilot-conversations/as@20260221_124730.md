### 链栈的定义

采用链式存储结构实现的栈称为链栈。链栈的优点是便于动态分配存储空间，不存在栈满溢出的问题，并且在多栈共享场景下能更灵活地利用内存。

### 结构体定义

```c++
typedef struct LinkNode{
    ElemType data;            // 数据域
    struct LinkNode *next;    // 指针域
} LiStack;                     // 链栈类型定义
```

### 链栈的优势

由于采用链式存储，结点的插入与删除操作非常高效。链栈的操作类似于单链表，入栈和出栈均在表头进行。

### 其他注意事项

- 对于带头结点的链栈，其初始化、判空及操作细节会有所不同。
- 本节所述实现基于无头结点的设计，读者应根据实际需求灵活调整实现方式。

2. **空栈判断**：在进行入栈或出栈操作前，需要检查栈是否为空。如果栈为空，则不能执行出栈操作。

3. **入栈操作**：
   - 将新结点插入到链表的头部。
   - 更新栈顶指针 `Lhead` 指向新的栈顶元素。

4. **出栈操作**：
   - 删除链表头部的结点。
   - 更新栈顶指针 `Lhead` 为下一个结点（即原栈顶结点的后继）。

5. **内存管理**：在进行入栈和出栈操作时，注意释放不再需要的结点空间，避免内存泄漏。

6. **边界条件处理**：
   - 在链表为空的情况下执行出栈操作会引发错误。
   - 确保所有指针操作正确无误，防止空指针引用等问题。

7. **多栈共享场景下的灵活性**：在实际应用中，多个链栈可能需要共用同一段内存空间。确保每个链栈的操作不会影响其他链栈的状态。
