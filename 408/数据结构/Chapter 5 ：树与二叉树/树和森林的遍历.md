### 树和森林的遍历

#### 树的遍历

树的遍历是指以某种方式访问树中的每个结点，且仅访问一次。主要有两种方式：

1. **先根遍历**：
   - 若树非空，则按如下规则遍历：
     1. 先访问根结点。
     2. 再依次遍历根结点的每棵子树，遍历子树时仍遵循先根后子树的规则。  
     其遍历序列与这棵树相应二叉树的先序序列相同。

2. **后根遍历**：
   - 若树非空，则按如下规则遍历：
     1. 先依次遍历根结点的每棵子树，遍历子树时仍遵循先子树后根的规则。
     2. 再访问根结点。  
     其遍历序列与这棵树相应二叉树的中序序列相同。

图 5.22 的树的先根遍历序列为 $ABEFCDG$，后根遍历序列为 $EFBCGDA$。此外，树也支持**层次遍历**，与二叉树的层次遍历原理相似，即按层序依次访问各个结点。

#### 森林的遍历

基于森林和树相互递归的定义，可得出森林的遍历方法：

1. **先序遍历森林**：
   - 若森林为非空，则按如下规则遍历：
     1. 访问森林中第一棵树的根结点。
     2. 先序遍历第一棵树中根结点的子树森林。
     3. 先序遍历除去第一棵树后的剩余森林。

2. **中序遍历森林**：
   - 若森林为非空，则按如下规则遍历：
     1. 中序遍历第一棵树中根结点的子树森林。
     2. 访问森林中第一棵树的根结点。
     3. 中序遍历除去第一棵树后的剩余森林。

图 5.23 的森林的先序遍历序列为 $ABCDEFGHI$，中序遍历序列为 $BCDAFEHIG$。

#### 树和森林的层次遍历

1. **层次遍历**：
   - 层次遍历是按从上到下、从左到右的顺序遍历所有结点。
   具体步骤如下：
     1. 使用一个队列来存储待处理的结点。
     2. 将当前树的根节点加入队列。
     3. 当队列不为空时，重复以下操作：
        - 从队列中取出一个结点并访问它。
        - 如果该结点有左孩子，则将左孩子加入队列。
        - 如果该结点有右孩子，则将右孩子加入队列。

#### 遍历关系

当森林转换成二叉树时，第一棵树的子树森林会转换为左子树，剩余森林则转换为右子树。因此，森林的先序、中序遍历与其对应二叉树的先序、中序遍历相同。

| 树    | 森林   | 二叉树  |
| ---- | ---- | ---- |
| 先根遍历 | 先序遍历 | 先序遍历 |
| 后根遍历 | 中序遍历 | 中序遍历 |
|      |      |      |
