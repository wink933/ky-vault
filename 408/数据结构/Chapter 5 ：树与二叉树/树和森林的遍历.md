### 树和森林的遍历

#### 树的遍历

树的遍历是指以某种方式访问树中的每个结点，且仅访问一次。主要有两种方式：

1. **先根遍历**：
   - 若树非空，则按如下规则遍历：
     1. 先访问根结点。
     2. 再依次遍历根结点的每棵子树，遍历子树时仍遵循先根后子树的规则。  
     其遍历序列与这棵树相应二叉树的**先序序列**相同。

2. **后根遍历**：
   - 若树非空，则按如下规则遍历：
     1. 先依次遍历根结点的每棵子树，遍历子树时仍遵循先子树后根的规则。
     2. 再访问根结点。  
     其遍历序列与这棵树相应二叉树的**中序序列**相同。
2. **层次遍历**：
   - 层次遍历是按从上到下、从左到右的顺序遍历所有结点。
   **具体步骤**如下：
     1. 使用一个队列来存储待处理的结点。
     2. 将当前树的根节点加入队列。
     3. 当队列不为空时，重复以下操作：
        - 从队列中取出一个结点并访问它。
        - 如果该结点有左孩子，则将左孩子加入队列。
        - 如果该结点有右孩子，则将右孩子加入队列。

#### 森林的遍历

基于森林和树相互递归的定义，可得出森林的遍历方法：

1. **先序遍历森林**：
   - 若森林为非空，则按如下规则遍历：
     1. 访问森林中第一棵树的根结点。
     2. 先序遍历第一棵树中根结点的子树森林。
     3. 先序遍历除去第一棵树后的剩余森林。

2. **中序遍历森林**：
   - 若森林为非空，则按如下规则遍历：
     1. 中序遍历第一棵树中根结点的子树森林。
     2. 访问森林中第一棵树的根结点。
     3. 中序遍历除去第一棵树后的剩余森林。

#### 遍历关系

当森林转换成二叉树时，第一棵树的子树森林会转换为左子树，剩余森林则转换为右子树。因此，森林的先序、中序遍历与其对应二叉树的先序、中序遍历相同。

| 树    | 森林         | 二叉树  |
| ---- | ---------- | ---- |
| 先根遍历 | 先序遍历       | 先序遍历 |
| 后根遍历 | 中序遍历（后序遍历） | 中序遍历 |
|      |            |      |
#### 易错点 / 易混点辨析
#####  “后根/后序”对应“中序”的超级陷阱

   ◦ **易错点**：做选择题时，极易想当然地认为树的“后根遍历”对应二叉树的“后序遍历”。

   ◦ **辨析**：**树的后根遍历必定对应二叉树的中序遍历！** 因为在“左孩子右兄弟”的结构下，树的子结点全部挂在左子树连缀上，兄弟结点全挂在右子树连缀上。树的后根遍历（子树 → 根），在二叉树看来就是（左子树 → 根 → 右子树），即中序遍历。
   
##### 森林遍历的命名分歧

   ◦ **易混点**：题目中可能会突然出现“对森林进行中序遍历”的字眼。

   ◦ **辨析**：由于森林转化为二叉树后的中序遍历，等同于森林的后序遍历，严蔚敏老师的教材中直接将森林的后序遍历称为**森林的中序遍历**。在408考试中，如果看到“森林的中序遍历”，直接将其等价替换为“转化为二叉树后的中序遍历”或“森林的后序遍历”来做题即可。
