## 内部排序算法的比较

从五个方面进行对比：**时间复杂度、空间复杂度、稳定性、适用性和过程特征**。
再补充两个方面，分别是**元素比较次数**以及**元素移动次数**
### 从时间复杂度看
简单选择排序、直接插入排序和冒泡排序在平均情况下的时间复杂度均为 $O(n^2)$，且实现较为简单。其中，直接插入排序和冒泡排序在最好情况下可达 $O(n)$，而简单选择排序的时间复杂度与序列初始状态无关。希尔排序作为插入排序的改进，对较大规模的数据通常具有较高效率，但目前未知其精确的渐近时间。堆排序利用堆这种数据结构，可在 $O(n)$ 时间内完成建堆，整个排序过程的时间复杂度为 $O(n\log_2n)$。快速排序基于分治思想，最坏情况下的时间复杂度为 $O(n^2)$，但平均性能可达 $O(n\log_2n)$，在实际应用中快速排序通常表现优异。归并排序同样基于分治思想，其分割过程与初始序列无关，因此最好、最坏和平均时间复杂度均为 $O(n\log_2n)$。
1. **坚如磐石的** O(nlog2​n) **梯队（不受初始状态影响）**
	- **堆排序、2路归并排序**：
	    - **表现**：这两种算法堪称“最稳的高级排序”，无论是最好、平均还是最坏情况，时间复杂度**统统死死钉在** O(nlog2​n)。
	    - **原因**：归并排序的二路划分递归树和堆排序的完全二叉树结构极其稳定，不因数据本来就有序还是无序而改变合并或调整的树高。

2. **极度依赖初始状态的“双刃剑”（最好** O(n) **或最坏退化）**

	- **快速排序**：
	    - **表现**：平均和最好情况为 O(nlog2​n)，但**最坏情况会暴跌至** O(n2)。
	    - **原因**：当初始序列已经基本有序或基本逆序时，快排的枢轴划分会极度不平衡（退化成单链表），导致时间复杂度最差。但在完全随机杂乱的情况下，快排是所有内部排序中平均性能最优的。
	- **直接插入排序、冒泡排序**：
	    - **表现**：平均和最坏情况均为 O(n2)，但**最好情况可达** O(n)。
	    - **原因**：当初始序列已经**基本有序**时，直接插入排序只需比较 1 次且不移动，冒泡排序在一趟无交换后即可提前终止，时间开销均锐减至 O(n)。

3. **“雷打不动”的** O(n2) **摆烂选手**

	- **简单选择排序**：
	    - **表现**：无论初始序列是正序、逆序还是乱序，最好、平均、最坏时间复杂度**全部都是** O(n2)。
	    - **原因**：它极其“死脑筋”，即使数组已经排好序，它依然要在无序区老老实实地从头比到尾找最小元素。元素比较次数固定为 n(n−1)/2，与初始状态毫无关系。

4. **特殊梯队（希尔与基数）**

	- **希尔排序**：时间复杂度高度依赖于“增量序列”的选取规则，最坏情况退化为 O(n2)，当 n 在某个范围内时约为 O(n1.3)。
	- **基数排序**：作为非比较类排序，时间复杂度为线性的 O(d(n+r))（d为趟数/位数，n为元素个数，r为基数/队列数），同样且不受初始序列顺序状态的影响。
### 从空间复杂度看
简单选择排序、插入排序、冒泡排序、希尔排序和堆排序仅需常数级辅助空间，空间复杂度为 $O(1)$。快速排序需要递归工作栈，平均空间复杂度为 $O(\log_2n)$，最坏情况下可能达到 $O(n)$。2 路归并排序在合并操作中需要额外的辅助数组用于元素复制，空间复杂度为 $O(n)$。虽有方法可减少辅助空间，但通常会导致算法复杂度显著增加，甚至影响时间效率。
1. **** O(1) **梯队：原地工作（绝对的主力）**

	- **包含算法**：直接插入排序、折半插入排序、冒泡排序、简单选择排序、希尔排序、**堆排序**。
	- **原因**：这些算法在执行比较和移动元素时，仅需要常数个辅助单元（例如用于暂存待插入元素的哨兵、用于交换元素的临时变量等），额外空间与问题规模 n 无关。

2. **** O(log2​n)∼O(n) **梯队：动态变化的系统栈空间**

	- **包含算法**：**快速排序**。
	- **原因**：快速排序采用分治法，高度依赖**递归调用**，因此需要借助系统栈来保存各层递归的上下文信息（如首尾指针）。
	- **空间变化规律**：
	    - **最好/平均情况**：划分均匀，递归树深度为 log2​n，空间复杂度为 O(log2​n)。
	    - **最坏情况**：初始序列基本有序或逆序，划分极度不平衡（退化为单链表），递归树深度达到 n，空间复杂度暴增至 O(n)。

3. **O(n) **梯队：等比例的辅助数组**

	- **包含算法**：**2路归并排序**。
	- **原因**：在执行 `Merge`（合并）操作时，必须将原数组中的两个有序段复制到一个**与原数组大小相同**的临时辅助数组中进行比对合并。因此，需要严格消耗 O(n) 的额外空间。

4. **O(r) **梯队：按基数分配的队列指针**

	- **包含算法**：**基数排序**。
	- **原因**：基数排序需要进行 d 趟分配与收集。在每趟分配时，需要为 r 个桶（即基数，例如十进制数字的基数 r=10）维护**头指针和尾指针**。这些指针在后续趟中可重复使用，因此空间复杂度为固定的 O(r)，与数据量 n 无关。
### 从稳定性看
插入排序、冒泡排序、归并排序和基数排序是稳定的排序算法，而简单选择排序、快速排序、希尔排序和堆排序都是不稳定的。在平均时间复杂度为 $O(n\log_2n)$ 的排序算法中，只有归并排序是稳定的。判断排序算法是否稳定，关键在于分析其操作是否会改变相等元素的相对次序；对于不稳定的算法，通常可通过构造一个反例加以验证。

#### 举例证明稳定性
1. **稳定的排序算法（核心在于“相邻比较”或“按序分配”）**
	- **直接/折半插入排序**：每次插入元素时，总是从后向前先比较再移动。当遇到相同的关键字时，会将其插入到该相同元素的**后面**，相对位置不发生变化。
	- **冒泡排序**：比较相邻元素时，判定条件严格为 A[i]>A[i+1] 才交换。遇到相等元素时不进行交换，因此是稳定的。
	- **归并排序**：在 `Merge` 合并两个有序子表时，当左、右子表出现相等元素时，算法会**优先将左子表**中的元素放入合并后的数组中，不会改变相对次序。
	- **基数排序**：每一趟基于位权（如个位、十位）的分配和收集都是严格**从前向后**进行的（采用队列 FIFO 思想），天然保持原有的相对顺序。

2. **不稳定的排序算法（核心在于“长距离跳跃交换”）**
	- **希尔排序**：由于采用了增量 d 进行分组，相同关键字的记录可能被划分到**不同的子表**中独立进行插入排序，这极易改变它们原本的相对次序。
	- **简单选择排序**：在无序区中找到最小元素后，会直接与无序区的“第一个元素”发生跨越式互换，如果“第一个元素”刚好和中间的某个元素同值，稳定性就会被破坏。
	- **快速排序**：在划分（Partition）过程中，左右指针交替扫描并交换元素。如果右端有两个相同的关键字且都小于基准值，它们被交换到左端后，相对位置会颠倒。
	- **堆排序**：在建堆或不断输出堆顶并进行向下调整（筛选）的过程中，位于树底层的相同关键字有可能被向上调整到树的不同层级，从而导致相对位置改变。

### 从适用性看
折半插入排序、希尔排序、快速排序和堆排序适用于顺序存储。直接插入排序、冒泡排序、简单选择排序、归并排序和基数排序既适用于顺序存储，也适用于链式存储。
1. **基于“底层存储结构”的选择（408 极高频考点）**

	- **仅适用顺序存储（依赖随机存取）**：折半插入排序、希尔排序、快速排序、堆排序。这些算法在执行过程中需要通过下标进行大跨度的元素跳跃访问或计算中间位置。
	- **顺序与链式存储均适用**：直接插入排序、冒泡排序、简单选择排序、归并排序、基数排序。若底层采用链表，还可以避免因频繁移动记录而产生的高昂开销。

2. **基于“数据规模（**n**）”的选择**

	- **当** n **较小（如** n≤50**）时**：
	    - 推荐使用**直接插入排序**或**简单选择排序**。
	    - 若每个元素包含的数据量很大（即移动元素的代价很高），优先选用**简单选择排序**，因为它的元素移动次数最少。
	- **当** n **较大时**：应选用时间复杂度为 O(nlog2​n) 的高级排序算法。
	    - **快速排序**：平均性能最优，是目前基于比较的内部排序中最好的方法。适用于绝大多数随机杂乱的大规模数据。
	    - **堆排序**：辅助空间极小（O(1)），且最坏情况下也能保证 O(nlog2​n)，适用于内存受限的场景或求“Top-K（前 k 个最大/最小元素）”问题。
	    - **归并排序**：当既要求极快的时间复杂度（O(nlog2​n)），又要求**稳定**时，归并排序是唯一选择，但代价是需要 O(n) 的辅助空间。

3. **基于“初始状态”的选择**

	- 若初始序列已**基本有序**：优先选用**直接插入排序**或**冒泡排序**。在这两种状态下，它们的时间复杂度可逼近甚至达到最好的 O(n)。
	- _反向避雷_：若初始序列已经基本有序或基本逆序，**绝对不要用快速排序**，因为这会使其退化为最坏情况，时间复杂度暴跌至 O(n2)。

4. ** 针对“特定数据特征”的特化选择**

	- 如果数据 n 很大，但**关键字的范围较小且容易划分位权**（例如范围在 0~9999 的整数，或较短的英文字符串）：优先选用**基数排序**。它可以达到惊人的线性时间复杂度 O(d(n+r))。

### 从过程特征看
不同排序算法在一趟或几趟处理后的中间结果通常是不同的。考研题目常给出初始序列和部分排序后的结果，要求判断所用的排序方法。这就要求考生熟练掌握各类算法的过程特征。如冒泡排序、简单选择排序和堆排序在每趟处理后都能确定当前最大（或最小）元素的最终位置，而快速排序在一趟划分后，至少能确定一个元素（枢轴）的最终位置等。

#### 过程特征说明
1. **每一趟必然能确定至少一个元素最终位置的算法**

	- **冒泡排序、简单选择排序、堆排序**：这三种算法在每一趟处理后，都能确定当趟最大（或最小）元素的最终位置。
	    - _特征表现_：序列的头部或尾部会出现逐渐增长的、绝对就位的“有序极值区”。
	- **快速排序**：在每一趟（一次划分）后，至少能确定一个元素（即枢轴 pivot）的最终位置。
	    - _特征表现_：枢轴左侧的所有元素均小于枢轴，右侧的所有元素均大于枢轴。

2. **每一趟能产生局部有序，但不能确定最终位置的算法**

	- **直接插入排序、折半插入排序**：每一趟处理后，前 i 个元素必定是有序的。
	    - _特征表现_：序列的前缀形成一个相对有序的子序列，但这些元素仍有可能在后续操作中被向后移动，因此它们不在最终位置。
	- **2路归并排序**：每一趟处理结束后，都会形成若干个长度为 2k 的相邻有序子序列。
	    - _特征表现_：呈现出明显的“块状有序”（如两两有序、四四有序）。

3. **呈现跳跃式局部有序的算法**

	- **希尔排序**：每一趟处理使得相隔一定“增量”的元素构成有序子序列。只有当最后一次增量为 1 时，才完成全局排序。
	    - _特征表现_：序列整体看起来仍是乱序的，但抽取相隔固定步长的元素看，是局部有序的。

### 从元素比较次数来看

#### 关键点（按比较次数特性分类归纳）

**1. 比较次数【与初始状态绝对无关】的算法**

- **简单选择排序**：
    - **结论**：比较次数始终严格为 n(n−1)/2 次,。
    - **原因**：无论数组是否有序，它每一趟都必须在无序区中从头扫描到尾去找最小元素，没有任何提前终止机制,。
- **折半插入排序**：
    - **结论**：比较次数约为 O(nlog2​n)，且**比较次数与序列初态无关**,。
    - **原因**：它利用折半查找来确定插入位置，折半查找的比较次数仅取决于表长 n，与待排序表的初始状态无关,。

**2. 比较次数【高度依赖初始状态】的算法**

- **直接插入排序 & 冒泡排序**：
    - **最好情况（初始为正序）**：只需进行 n−1 次比较,。直接插入排序每次插入只需与前一个元素比 1 次；冒泡排序第一趟比较 n−1 次后发现无交换，直接结束,。
    - **最坏情况（初始为逆序）**：都需要进行 n(n−1)/2 次比较,。
- **快速排序**：
    - **表现**：快速排序的比较次数同样依赖于初始序列的状态。最坏情况下（初始序列基本有序或逆序），每次划分极度不平衡，比较次数退化至 O(n2) 量级。

**3. 特殊操作场景的精准比较次数（常考综合题结论）**

- **堆排序的建堆操作**：在含有 n 个元素的序列上建立初始堆时，关键字的总比较次数**不超过** 4n,。
- **2 路归并排序的合并操作**：将两个长度分别为 m 和 n 的有序表合并成一个有序表，最少需要比较 min(m,n) 次，**最多需要比较** m+n−1 **次**。

**4. 比较次数为【 0 】的算法**

- **基数排序**：作为典型的非比较类排序，它完全依靠将关键字拆分为不同位进行“分配”和“收集”，全程不需要进行关键字的比较,。

#### 表格总结

| 排序算法       | 最好情况比较次数（正序） | 最坏情况比较次数（逆序） | 比较次数是否依赖初始状态？ |
| ---------- | ------------ | ------------ | ------------- |
| **直接插入排序** | n−1          | n(n−1)/2     | **是**         |
| **冒泡排序**   | n−1,         | n(n−1)/2     | **是**         |
| **简单选择排序** | n(n−1)/2     | n(n−1)/2     | **否（雷打不动）**   |
| **折半插入排序** | O(nlog2​n)   | O(nlog2​n)   | **否**,        |
| **基数排序**   | 0            | 0            | **否（非比较类）**   |

### 从移动次数来看

#### 关键点（按移动次数特性分类归纳）

**1. 移动次数【与初始状态绝对无关】的算法**

- **基数排序**：
    - **结论**：元素的移动次数与序列初态绝对无关。
    - **原因**：它完全按位权进行硬性的“分配”和“收集”，无论初始序列是正序还是逆序，每一趟都必定要将所有有效元素依次分发到对应队列，再依次从队列中收回原数组。因此它的数据移动轨迹是固定且统一的。

**2. 移动次数极低，最高不超过** O(n) **的算法**

- **简单选择排序**：
    - **结论**：最好情况下移动 0 次，最坏情况下总移动次数最多不超过 3(n−1) 次。
    - **原因**：它每一趟在无序区找到最小元素后，最多只进行 1 次跨越式交换（1 次交换对应 3 次元素的移动）。因此，它是所有基础比较排序中，**应对“元素数据量大、移动极其耗时”场景的最优解**。

**3. 移动次数【高度依赖初始状态】的算法**

- **直接插入排序 & 冒泡排序**：
    - **最好情况（正序）**：移动次数均为 **0**。
    - **最坏情况（逆序）**：移动次数均暴增至 O(n2) 量级。例如冒泡排序在最坏情况下的移动次数高达 23n(n−1)​ 次（每次比较后都需要 3 次移动来完成交换）。
- **折半插入排序**：
    - **结论**：元素的移动次数与直接插入排序完全相同，依然依赖于初始序列状态，最坏仍是 O(n2)。
    - **原因**：它仅仅优化了“找插入位置”的比较过程，但并没有改变“腾出空间”时需要大面积挪动元素的物理机制。

| **直接插入排序** | **0**        | O(n2)               | **是**         | 移动次数与初始状态有关。 |
| **冒泡排序**   | **0**        | $\frac{3n(n-1)}{2}$ | **是**         | 最好情况0移动。     |
| **折半插入排序** | 0            | O(n2)               | **是**         | **移动次数未减少**。 |
| **简单选择排序** | **0**        | **最多** 3(n−1)       | **是**         | 移动数据极其“抠门”。  |
| **基数排序**   | 与 d 和 n 有关   | 与 d 和 n 有关          | **绝对无关**      | 唯一移动次数无关的算法。 |


## 8.6.2 内部排序算法的应用

通常情况，对排序算法的比较和应用应考虑以下情况。


#### 1. 选取排序算法需要考虑的因素

1. 待排序元素的个数 $n$。
    
2. 待排序序列的初始状态（如是否基本有序）。
    
3. 关键字的结构及其分布情况。
    
4. 对稳定性的要求。
    
5. 存储结构类型及辅助空间的限制等。
    

#### 2. 排序算法选型小结

1. 若 $n$ 较小，可采用直接插入排序或简单选择排序。直接插入排序的记录移动次数通常多于简单选择排序，因此当记录本身信息量较大时，简单选择排序更为合适。
    
2. 若 $n$ 较大，应采用时间复杂度为 $O(n\log_2n)$ 的排序算法，如快速排序、堆排序或归并排序。当关键字随机分布时，快速排序通常被认为是最高效的基于比较的内部排序算法。堆排序所需辅助空间少于快速排序，且不会退化到最坏情况，但两者均为不稳定排序。若既要求稳定性，又希望达到 $O(n\log_2n)$ 的时间复杂度，则应选用归并排序。
    
3. 若初始序列已基本有序，则直接插入或冒泡排序更为高效，因其最好时间复杂度为 $O(n)$。
    
4. 在基于比较的排序算法中，每次比较两个关键字仅有两种可能的结果，因此其执行过程可用一棵判定树来描述。由此可证明：对于 $n$ 个关键字的随机排列，任何基于比较的排序算法，在最坏情况下至少需要 $O(n\log_2n)$ 的时间。
    
5. 若 $n$ 很大，且关键字位数较少、可分解为多个子关键字（如整数、字符串），则基数排序是更优选择，因其时间复杂度可接近线性。
    
6. 当记录本身信息量较大（如包含大量字段或大对象）时，为避免因频繁移动记录而产生高昂开销，宜采用链表作为存储结构，此时插入类或归并类排序更具优势。
