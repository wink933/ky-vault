# Markdown 超详细新手指南（Obsidian 版）

> 目标：学会“写笔记需要的 99% Markdown”，并能在 Obsidian 里写出**清晰、结构化、可复用**的笔记。

---

## 0. Markdown 是什么，你为什么要学它

- **Markdown = 轻量标记语言**：用少量符号（#、-、`）就能排版。
    
- 你得到的好处：
    
    - **写得快**：不用鼠标点格式
        
    - **结构清晰**：标题、列表、引用、代码一目了然
        
    - **可迁移**：纯文本，放 Git、云盘、任何编辑器都不怕
        
    - **Obsidian 加成**：链接、标签、Callout、脚注、公式、代码块高亮
        

---

## 1. 基础排版：段落、换行、空行（新手最容易错）

### 1.1 段落

- **空一行**表示新段落：
    

```md
这是第一段。

这是第二段。
```

### 1.2 换行（同一段内换到下一行）

在行尾加 **两个空格** 再回车，或者用 HTML 的 `<br>`：

```md
第一行（后面有两个空格）  
第二行
```

或：

```md
第一行<br>
第二行
```

---

## 2. 标题：笔记骨架（最重要）

用 `#` 表示标题级别：

```md
# 一级标题（整篇文章标题）
## 二级标题（大章节）
### 三级标题（小节）
#### 四级标题
```

**建议（Obsidian 常用）**

- 一篇笔记一般只用到 `# ~ ###`
    
- 标题尽量像目录一样“可扫读”
    

---

## 3. 强调：加粗、斜体、删除线、高亮

```md
**加粗**
*斜体*
~~删除线~~
==高亮==   （Obsidian 支持）
```

效果：

- **加粗**
    
- _斜体_
    
- ~~删除线~~
    
- ==高亮==
    

---

## 4. 列表：最常用结构

### 4.1 无序列表（- 或 * 都行）

```md
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2
```

### 4.2 有序列表

```md
1. 第一步
2. 第二步
3. 第三步
```

### 4.3 任务清单（Obsidian 超常用）

```md
- [ ] 未完成
- [x] 已完成
```

你还可以做“学习计划/打卡”：

```md
- [ ] 背诵：TCP 三次握手
- [ ] 刷题：子网划分 10 题
- [x] 总结：ARP 易错点
```

## 5. 引用：做“解释、原话、结论”特别好用

```md
> 这是一段引用内容
> 可以写多行
```

嵌套引用：

```md
> 外层引用
>> 内层引用
```

## 6. 分割线：做章节分隔/总结区

```md
---
```

常用于“正文 / 总结 / 练习题”分割。

---

## 7. 代码：行内代码 & 代码块（理工科必备）

### 7.1 行内代码（1 行里突出某个词）

用一对反引号 `` ` ``：

```md
TCP 使用 `滑动窗口` 进行流量控制。
```

### 7.2 代码块（多行）

用三对反引号：

````md
```python
def add(a, b):
    return a + b
```
````

常见语言标注：`c` `cpp` `python` `java` `js` `ts` `bash` `json` `yaml` `sql`

---

## 8. 链接：外部链接 & Obsidian 内部链接

### 8.1 外部链接

```md
[显示文字](https://example.com)
```

### 8.2 Obsidian 内部链接（知识网络核心）

```md
[[TCP]]
[[计算机网络/传输层/TCP]]
```

带显示名（别名）：

```md
[[TCP|传输控制协议（TCP）]]
```

链接到某个标题：

```md
[[TCP#三次握手]]
```

---

## 9. 图片：插图、截图、课件

### 9.1 插入图片（Obsidian 常见）

```md
![[image.png]]
```

### 9.2 标准 Markdown 图片（也能用）

```md
![说明文字](image.png)
```

建议你把图片放在固定目录，例如：

- `assets/`
    
- `images/`
    
- `attachments/`

## 10. 表格：对比、总结神器（考试必备）

基础表格：

```md
| 协议 | 是否可靠 | 是否面向连接 | 典型场景 |
|------|----------|--------------|----------|
| TCP  | 是       | 是           | Web/文件 |
| UDP  | 否       | 否           | 直播/游戏 |
```

提示：

- 对齐不影响渲染，但对齐更好看
    
- 表格里可以放加粗、代码：`**重点**`、`TCP`

## 11. Callout（Obsidian 特色）：做“重点/警告/例题/总结”

语法：

```md
> [!note] 笔记
> 这是一个 note callout
```

常用类型（推荐你背下来）：

```md
> [!info] 信息
> [!tip] 技巧
> [!success] 结论
> [!warning] 易错点
> [!danger] 坑
> [!example] 例子
> [!question] 问题
> [!summary] 总结
```

可折叠：

```md
> [!summary]+ 可展开的总结
> 内容...
```

把 `+` 改成 `-` 表示默认折叠：

```md
> [!summary]- 默认折叠
> 内容...
```

这是一个观点。[^1]

[^1]: 这里写脚注内容或来源说明。
```

## 16. Obsidian 里“超实用”的 Markdown 扩展技巧

### 16.1 YAML Front Matter（可选）

放在文件最顶部，用来写元信息（不影响正文阅读）：

```md
---
topic: TCP
module: 传输层
tags: [408, 计网]
created: 2026-01-22
---
```

用途：

- 配合 Dataview 等插件做“索引、统计、自动汇总”
    
- 做错题库/知识库元数据管理
    

### 16.2 引用块里放代码（常用于“题目 + 解答”）

```md
> 题目：解释为什么 UDP 不可靠
>
> ```text
> 因为 UDP 无连接、无重传、无序号确认...
> ```
```

### 16.3 快速做“模块化笔记模板”

你可以把常用结构固定为：

```md
## 【本质】
## 【要解决的问题】
## 【机制链条】
## 【状态机/流程】
## 【代价分析】
## 【场景适用性】
## 【对比与替代机制】
## 【真题考法与趋势】
## 【易错点】
## 【用于 Obsidian 固化】
```

## 17. 常见坑（新手必看）

1. **列表缩进必须一致**  
    子列表建议用两个空格或四个空格，不要乱跳。
    
2. **段落要空行**  
    不空行会挤成一团。
    
3. **代码块反引号要成对**  
    少一个就会导致后面全变成代码。
    
4. **表格的竖线要对**  
    少一个 `|` 直接渲染失败。
    
5. **Obsidian 内链不要写成 URL**  
    直接 `[[文件名]]` 最方便。

## 18. 一份“你可以直接复用”的 Obsidian 笔记示例（计网版）

```md
---
topic: TCP 三次握手
module: 传输层
tags: [408, 计网, TCP]
---

# TCP 三次握手

> [!summary] 一句话
> 用最小的报文交换完成“双方收发能力确认 + 初始序号同步 + 连接参数建立”。

## 【本质】
- 建立可靠连接的**状态同步协议**

## 【机制链条】
1. Client 发送 SYN（带初始序号 x）
2. Server 回 SYN+ACK（带初始序号 y，确认 x+1）
3. Client 回 ACK（确认 y+1）

## 【易错点】
> [!warning]
> - 第二次握手是 SYN+ACK，不是 ACK
> - 确认号是“对方序号 + 1”

## 【对比】
| 项目 | 两次握手 | 三次握手 |
|------|----------|----------|
| 是否能确认双方收发能力 | 否 | 是 |
| 是否避免历史连接干扰 | 弱 | 强 |
```
