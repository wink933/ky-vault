# git与github的关系

## 直觉类比

- **Git** = 你电脑上的“版本控制引擎”（像 Word 的“版本历史”，但更强、可协作、适合代码/文档）
    
- **GitHub** = 一个“托管你 Git 仓库的云平台”（像网盘 + 协作平台 + 项目管理 + 自动化）

## 各自分工

### Git 管的是：**历史（History）**

Git 在你本地项目目录里创建一个隐藏文件夹：`.git/`  
里面保存了：

- 每次提交（commit）的**完整快照**
    
- 提交之间的**父子关系**（一条历史线/树）
    
- 分支指针、HEAD 指针
    
- 文件内容对象（blob/tree/commit 等）
    

所以：  
✅ **只要你 `git commit` 了，即使不联网，你也有完整历史**  
✅ 你可以随时 `git log` 看历史，`git diff` 看改动，`git restore/revert` 回滚

---

### GitHub 管的是：**远端仓库 + 协作系统**

GitHub 上放的，是你本地仓库的一份“远程副本”（remote repository）。此外它还提供：

- **Pull Request（PR）**：把分支改动合并到主分支的“审核流程”
    
- **Issues**：任务/bug/讨论跟踪
    
- **Actions**：CI/CD 自动化（测试、构建、部署）
    
- **权限管理**：谁能读/写/审核
    
- **代码浏览、Wiki、Releases** 等
    

所以：  
✅ GitHub 让你“备份 + 多设备同步 + 多人协作”变得标准化  
❌ 但 GitHub 不是 Git 的一部分，Git 也不是 GitHub 专用
